## מה בונים

שני מסכי מנהל חדשים המחוברים לדשבורד, שניהם מבוססים בינה מלאכותית מעל נתוני השאלונים הקיימים.

---

### 1. תמונה כיתתית (Class Insights)

מסך שנפתח מהלשונית של כל כיתה ("הכיתה של טלי" / "עדן" / כל כיתה מותאמת) → כפתור **"תמונה כיתתית"**.

**מה מוצג:**
- **סטטיסטיקה מצטברת** — כמה השלימו, פילוח מגדר, טווח גיל, ממוצע כיתתי לכל תחום (איכות חיים, מסוגלות עצמית, שליטה, גמישות, למידה) עם השוואה לשאר הכיתות.
- **גרף חוזקות/חולשות רוחביות** — heat-map של הפריטים החוזרים אצל רוב הכיתה (רגישויות משותפות, קשיים משותפים, חוזקות משותפות).
- **תובנות בינה** (טקסט מובנה שהמודל מפיק):
  1. 3-5 נושאים שמעסיקים את רוב תלמידי הכיתה (מבוסס תשובות פתוחות + פריטים דומיננטיים).
  2. מוקד רוחבי מומלץ לטיפול הקבוצתי השבועי — נושא אחד ברור + 2-3 טכניקות/תרגילים.
  3. תת-קבוצות בתוך הכיתה (למשל: 4 תלמידים עם קושי חברתי דומה, 3 עם עומס חושי) — עם שמות התלמידים המרכיבים כל תת-קבוצה.
  4. תלמידים "עוגן" (חוזקות שיכולות לסייע לאחרים) ותלמידים "בסיכון מוגבר" בכיתה.

---

### 2. מנוע שיבוץ חכם (Placement Engine)

מסך נפרד מ־Dashboard → כפתור עליון **"מנוע שיבוץ"**.

**זרימה:**
1. המסך מציג את כל התלמידים ללא שיוך (ואופציונלית — גם משוייכים לצורך "רה־שיבוץ").
2. המשתמש בוחר תלמיד → לוחץ **"הצע שיבוץ"**.
3. הבינה מנתחת את פרופיל התלמיד מול הרכב כל אחת מהכיתות ומחזירה:
   - **המלצה** (איזו כיתה + רמת ביטחון: גבוה/בינוני/נמוך).
   - **רציונל מילולי מפורט** בעברית שכולל: התאמות (גיל/מגדר/מאפייני למידה/חוזקות/אתגרים), עם מי מהתלמידים הקיימים בכיתה יש חפיפה חיובית, איזה איזון הכיתה מקבלת (למשל: "בכיתה של עדן חסר איזון מגדרי, ולהב יוסיף...", "נעם חולק עם 3 תלמידים בכיתה של טלי קושי דומה בוויסות → יוצר עוגן קבוצתי לטיפול").
   - **אלטרנטיבה** — הכיתה השנייה הכי טובה + למה פחות מתאים.
   - **דגלים** — התראה אם השיבוץ יוצר עומס (למשל: "4 תלמידים עם סיכון גבוה בכיתה אחת").
4. כפתור **"אשר שיבוץ"** מעדכן את classGroup של התלמיד.

**מצב אצווה:** כפתור **"הצע שיבוץ לכל הלא־משוייכים"** מריץ את המנוע על כל התלמידים בבת אחת ומציג טבלת סיכום שניתן לאשר אחד־אחד או "אשר הכל".

### מבנה טכני

**Edge Functions חדשות** (Lovable Cloud, מודל: `google/gemini-2.5-pro`, טמפרטורה נמוכה):
- `class-insights` — קלט: classGroup + academicYear. אוסף את כל ה־sessions של הכיתה, מנקה את הנתונים, שולח למודל עם prompt מובנה בעברית → מחזיר JSON (נושאים, מוקד קבוצתי, תת־קבוצות, עוגנים, סיכונים).
- `placement-suggest` — קלט: studentId + רשימת כיתות. אוסף פרופיל התלמיד + פרופיל מצטבר של כל כיתה → מחזיר JSON (המלצה, ביטחון, רציונל, אלטרנטיבה, דגלים).

**קבצים חדשים בצד הלקוח:**
- `src/pages/admin/ClassInsights.tsx` — עמוד תובנות כיתתיות.
- `src/pages/admin/PlacementEngine.tsx` — מסך מנוע השיבוץ.
- `src/lib/class-aggregations.ts` — פונקציות אגרגציה מקומיות (ממוצעים, פילוח מגדר/גיל, זיהוי פריטים חוזרים) שמאכילות את ה־AI וגם את הגרפים.
- הוספת נתיבים ב־App.tsx וקישורים מ־Dashboard.

**עדכון סכימה:** הוספת שדה `gender` ל־`intake_sessions` (זכר/נקבה/לא צוין) — נחוץ לאיזון הכיתה. השדה יופיע גם בטופס הקליטה החדשה וגם כעריכה מהירה בפרופיל תלמיד.

**אחסון תובנות:** תובנות הכיתה נשמרות ב־`school_settings` (key: `class_insights_<classKey>_<year>`) עם timestamp, ומחושבות מחדש רק בלחיצה על "רענן תובנות" — כדי לחסוך קרדיטים.

### מה **לא** נכלל בשלב זה (יעלה בהמשך)

- **פרופיל מחנכת** — תואם להעדפתך "בהמשך". המנוע יציין רק "מומלץ לכיתה X" מבלי לבחור מחנכת. בשלב הבא נוסיף מסך "פרופילי מחנכות" (סגנון, חוזקות, סוגי ילדים שמתאימים לה) והמנוע ישקלל גם אותו.
- ייצוא PDF של תמונה כיתתית — אפשר להוסיף בסבב הבא.
